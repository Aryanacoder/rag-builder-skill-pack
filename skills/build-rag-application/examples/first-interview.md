# Example: First Interview

User:

> I want a RAG app for my company's manuals.

Agent:

> Good. I will turn this into a buildable RAG design before we choose code. First I need the shape of the problem:
>
> 1. What should users be able to decide or do after asking the app?
> 2. Who will use it, and what would be costly if the answer is wrong?
> 3. What data do you have: file types, languages, rough volume, scanned/native PDFs, tables or diagrams?
> 4. How often does the data change?
> 5. Must answers include citations, page numbers, previews, quotes, or downloads?
> 6. Does it need to run offline/local, in the cloud, or both?
> 7. What stack or idea do you already have in mind?
> 8. What first demo question must work?
